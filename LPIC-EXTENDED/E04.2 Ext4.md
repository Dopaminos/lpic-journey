# Структура файловой системы

Система организована с помощью блочных групп, где стандартный размер блока - 4КБ.

Блочные группы содержат в себе таблицы inode,  блоки данных, а также карта битов (что занято/свободно).
 
Такое устройство нужно для того, чтобы inode и данные были рядом, и, соответственно, время обращения к ним уменьшалось.

## Суперблок

Часть файловой системы, содержащая в себе
 - размер FS
 - размер блока
 - кол-во айнод и монтирований
 - время последней проверки системы
 - статус FS
Проверить содержимое суперблока можно с помощью
`tune2fs -l /dev/sdX` или `dumpe2fs -h /dev/sdX`

Суперблоки нужны для восстановления файловой системы, в случае ошибки.

## Таблица Inode

Таблица, содержащая в себе все доступны inode. 

Поскольку inode создаётся при создании файловой системы, inode закончиться легче, чем месту на накопителе.
## Карта битов

Показывает, что свободно, что занято, 0 и 1 по блокам соответственно. Карты битов есть не только по блокам, но и по inode-ам.

# Как создаётся файл

1. Выделяется inode под файл
2. Выделяется блок данных для файла
3. Каталог получает запись о выделенных ресурсах
4. Данные пишутся в нужную директорию

# Журналирование
