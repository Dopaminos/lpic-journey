Контрольные группы в Linux реализованы через sysfs, а именно `/sys/fs/cgroup`

В данной директории можно увидеть иерархию контрольных групп, где каждая отдельная директория является контрольной группой. 

В любую из существующих групп можно поместить процесс, из-за чего, на него наложатся ограничения конкретной группы.

пример tree с systemd:
```
/sys/fs/cgroup  
├── user.slice  
├── system.slice  
└── init.scope
```

В каждой директории есть файл `cgroup.procs`, где записываются PID процессов, принадлежащей какой-либо группе.

Чтобы перенести процесс в какую либо группу, нужно добавить его PID в этот файл, после чего, ограничения этой группы будут применены к процессу.


## Контроллеры

Ограничения ресурсов работают через .max файлы, например cpu.max, memory.max и т.д.

Некоторые примеры: 
```
echo "50000 100000" > cpu.max (<= 50% CPU)
echo 512M > memory.max (<= 512MB RAM)
echo 100 > pids.max (<= 100 PID)

```

## SystemD

SystemD автоматически создаёт контрольные группы для системы, пользователя и самой машины.

![](Pasted%20image%2020260309125214.png)

System слайс управляет системными демонами
(например, как на рисунке, крон, httpd, systemd)

User слайс нужен для пользовательских процессов (сессий), внутри которого, может быть деление на каждого юзера.

Machine слайс используется для выделения ресурсов контейнерам вроде докера и виртуалкам. 

## Полезные команды

`systemd-cgtop` - top по группам
`systemd-cgls` - ls групп
