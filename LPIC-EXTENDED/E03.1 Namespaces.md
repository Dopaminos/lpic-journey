
```
Пространство имён - механизм, обеспечивающий изоляцию глобальных ресурсов системы для группы процессов.
```

Процесс внутри namespace-a работает с **своей собственной** версией системных ресурсов, например, у процесса появляется :
- своя таблица процессов
- свои сетевые интерфейсы
- свой hostname
- своя таблица монтирования
  
И не смотря на вышеописанное, работа всех таковых процессов осуществляется внутри одного ядра.

# Типы пространств имён

Краткая сводка по всем типам namespace-ов описана ниже. 
Каждый тип будет рассмотрен отдельно в своих подпунктах.

| Namespace | Изолирует                                   |
| --------- | ------------------------------------------- |
| `pid`     | идентификаторы процессов                    |
| `net`     | сетевой стек                                |
| `mnt`     | точки монтирования                          |
| `uts`     | hostname и domain name                      |
| `ipc`     | механизмы IPC (inter-process communication) |
| `user`    | пользователей и UID/GID                     |
| `cgroup`  | иерархию cgroups                            |
## PID

Изолирует таблицу процессов.

С помощью этого namespace-a, процесс может видеть только процессы **внутри** своего пространства имён, например, хост имеет в себе:

- 1   systemd
- 222 nginx
- 333 postgres

но изолировав nginx, можно изменить таблицу процессов:

- 1 nginx
- 2 worker
- 3 worker

Важно, что PID 1 является init-процессом, принимающий на себя множество задач, из-за чего, часто, в контейнерах используют специальные init-процессы.

## Network

Изолирует сеть.

С помощью этого namespace-a, процесс имеет свой отдельный сетевой стек, где конфигурация может отличаться от хост-системы, благодаря чему, разные контейнеры могут использовать один и тот же порт из-за нахождения в разных namespace-ax.

Связь между namespace-ами осуществляется с помощью **veth** (виртуальный эзернет), где один конец интерфейса находится на контейнере, а другой на хосте

## Mount

Изолирует таблицу монтирования.

Создаёт, собственную для процесса, таблицу монтирования, даже если она отлична от хостовой.

## UTS

Изолирует имя системы и домена.

## IPC

Изолирует механизмы межпроцесcного взаимодействия, а именно:
-  общую память
-  очередь сообщений
-  семафоры

Что есть IPC - https://rus-linux.net/MyLDP/BOOKS/lpg-04/node6.htm
Вкратце: процесс может что-то сказать другому процессу и получить ответ.

Процессы в разных IPC namespace-ах не видят IPC объекты друг-друга.

## User

Позволяет изменить UID/GID, благодаря чему, контейнеры можно запускать не имея рута на хосте.


# Namespace на практике

Новый namespace можно создать с помощью `unshare`, например
```sh
unshare --fork --pid --mount-proc bash
```

Данная команда создаёт новый PID namespace и новую таблицу процессов и после её запуска, будут видны только процессы внутри namespace-а bash-a.

Смотреть namespace-ы можно с помощью `lsns`, 
а у конкретного процесса `ls -l /proc/<pid>/ns`
