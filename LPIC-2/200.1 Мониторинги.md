# CPU
## vmstat 
это утилита для мониторинга железа, показывает:
	 процессы (выполяемые >= колву ядер - не очень т.к. айдл, и блокируемые)
	 память (свап, свободное, буффер, кэш занято)
	 swap (выгружено, считано за интервал)
	 io (блочный ввод/вывод)
	 системные параметры (прерывания, переключения задач)
	 цпу (userspace%, system%, idle, wait%, stolen timeq)

## top (htop)
диспетчер задач частично с инфой из vmstat.

из интересного:
	 load average (самое важное)
	 uptime
	 кол-во user-ов
	 удобный просмотр столбцов в reatlime (ps aux)

# RAM

## vmstat
	swap - файл для подстраховки, который нужен при нехватки оперативной памяти. 
	free - свободно
	buffer - зарезервированная память под определённые процессы (heap)
	cache - %recent% / %temp%, данные, к которым надо часто обращаться
	si/so - выгружено / считано за интервал. 

vmstat -a добавляет параметры inact/active. неактивная память обычно свопится, активная используется программами. 

-s пишет всё в столбец, вместо строчки.

## free
vmstat, но только для памяти, 
	флаг -h преобразует значения в читаемые, в зависимости от объёма. 
	флаг -m преобразует значения в мегабайты.

## swapon
показывает своп. обычно используется флаг -s.


# HDD

## vmstat

для мониторинга блочных устройств, в vmstat есть секция io (ввод/вывод), где отображается их использование.

## iotop

top, но предназначенный сугубо для устройств ввода/выводы

он показывает:
	total i/o - пропускная способность между процессами и i/o
	actual i/o - показатель обращения программами к железу
	Thread ID - по нему можно найти PID и узнать, что грузит сервер
	prio - nice, но для io (0-7). be - best effort, rt - realtime
	user
	io%
	swapin% - процент, используемого программой, свопа   

## systat

пакет инструментов со статистикой.

iostat в себе имеет: 
	avg-cpu (всё с топа)
	Device - статистика по блочным устройствам.

из интересного там есть tps (запросы/обращения в секунду)

iostat -x -- очень подробный iostat имеющий информацию об: 
	обращениях r/w и в запросах и в кБ, 
	средний размер и длина запроса в секторах.
	await r/w (ожидание что-то сделать) в мС
	утилизация процессора в i/o

## sar

гигантская утилита, которая может выдать любой репорт о системе. 
без ключей даёт информацию по процу

-b блочное
-v verbose
-d дисковое (iostat)

# Network

## ifconfig 
как ipconfig на винде, но намного более подробная штука.

	mtu - maximum transmission unit (макс. размер блока полезных данных в пакете)
	inet - ip интерфейса
	netmask - маска
	broadcast - шлюз
	txqueuelen - длина очереди передачи
	TX/RX packets
TX/RX - Transmit / Receive

свойства ошибок пакетов:
	dropped - отброшенные (v6 пакеты при отключенном v6)
	overruns - fifo buffer error
	frame - не-целые кадры (не кратные 8, сбоит свич)
	carrier - проблемы несущей волны (кабель)
	collisions - несоответствия дуплекс-режимов между устройствами

scopeid - параметр известия нашего ip
	host - в пределах хоста
	site - v6
	link - локальная сеть
	global - интернет

## netstat

полезные флаги:
	-ie - ifconfig
	-s - статистика по протоколам
	-tuna - tcp/udp numeric all
	-tunlp - tcp/udp numeric listening pid
::: = 0.0.0.0 но ipv6

## iftop

top для интерфейса, показывает соединения, разные скорости.

## nload
красивые графики, уникального вывода нет

## vnstat

vmstat для сети, нужен для минимального использования трафика.
ключи:
	-m месяц
	-h час
	-d день

# Network Issues Diagnosis

## traceroute (tracert в винде) 
- выводит каждый шаг к заданному урлу, кинув 30 хопов пакетами по 60 байтов (как пример) со временем, на каждый хоп

флаг -T переключает режим трассировки с протокола ICMP на TCP
флаг -m меняет кол-во хопов
флаг -q меняет кол-во пакетов

`*` обозначает отсутствие ответа от маршрутизатора на хоп, не обязательно означает, что есть проблемы с сетью, может так же обозначать, что пункт подключения имеет файрвол блокирующий оба протокола
## mtr 

traceroute, но более подробный и обновляется в realtime
из подробного:
	потери пакетов в % и присланные пакеты
	вся статистика по пингам

флаг -P ставит порт
флаг -i ставит интервал в секундах
флаг -rwc не урезает имя доменов

# Bandwidth

## iperf3 / iperf
internet performance, 
флаг -с замеряет performance при подключении до сервера <ip>
флаг -n задаёт объём передаваемых данных
флаг -R позволяет замерить параметр с клиента на сервер (наоборот)
флаг -i задаёт интервал в секундах
показываются:
	Interval - время передачи
	Transfer - сколько передаём
	Bandwidth - пропускная способность канала 
	Retries - попытки отправить пакет по TCP


ethtool - показывает настройки интерфейса
