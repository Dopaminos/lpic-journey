# netstat -> ss

ss (socket statistics) - инструмент, позволяющий смотреть:
	- активные сетевые соединения
	- listening порты
	- tcp/udp сокеты и их статистику

почему ss, а не netstat?

ss работает напрямую с ядром через netlink API, благодаря чему, он может отобразить больше информации о сокетах.

полезные аргументы:
	-a      (всё)
	-t      (tcp)
	-tu     (tcp/udp)
	-l      (listening)
	-n      (numeric)
	-p      (процессы по pid)
	-e      (extended)
	-i      (tcp информация)
	-tulpen (процессы открывшие порт)
	-s      (статистика сокетов)
	
фильтрация вывода по порту может выполняться с помощью нескольких способов, среди которых часто используемыми являются
	встроенная фильтрация через = :port
	| grep :port

# ifconfig -> ip

ifconfig (interface config) использовался для:
	- просмотра доступных интерфейсов
	- назначения ip-адресов
	- вкл/выкл портов

ip объединяет в себе функциональность нескольких утилит, а именно:
	ifconfig
	route
	arp
	netstat

полезные аргументы:
	addr или a (просмотреть интерфейсы + ip на них)
	link (только интерфейсы)
	link set <интерфейс> up / down (поднять или вырубить интерфейс)
	route show (смотреть маршруты)
	neigh (bour, показывает arp таблицу)
	-s link (смотреть статистику интерфейса)
	addr add <ip> dev <interface> (+ip к уже имеющемуся (временно) )
	monitor (watch, сетевые изменения в realtime)
	-br (краткий вывод интерфейсов)
