```
Контрольные группы - механизм ограничения ресурсов и их управления/учёта между группами процессов.
```
По умолчанию, все процессы являются частью корневой группы, однако данный механизм позволять создавать "подгруппы" внутри любой ныне существующей группы, создавая иерархию, и перемещать процессы в эти подгруппы.

Далее, в соответствии с настройками контроллеров назначенных этой "иерархией", на группы накладываются ресурсные ограничения. 

```
Контроллер - подсистема ядра, отвечающая за конкретный тип ресурса.
```

Ограничить можно использование любого ресурса, у которого есть контроллер, хоть i/o, хоть cpu.

Существуют две версии механизма cgroups:

 - cgroups v1 - старый механизм, где каждый контроллер имеет собственную иерархию.

 - cgroups v2 - современный unified механизм, где все контроллеры работают в одной иерархии.

Современные системы Linux используют cgroups v2, поэтому дальнейшее рассмотрение будет опираться именно на него.
