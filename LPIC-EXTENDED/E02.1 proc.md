
```
/proc(procfs) является псевдо-файловой системой, содержащей в себе информацию о:
- состоянии системы
- запущенных процессах
- параметрах ядра

все данные, находящиеся в `/proc` генерируются в реальном времени.
```
## Важные файлы системы

### /proc/meminfo
файл с информацией о доступной памяти системы, содержит в себе: 
- MemTotal
- MemFree
- Buffers
- Cached
- SwapTotal
- SwapFree

этот файл используется `free`, `vmstat`, `top` и его вариациями.

### `/proc/stat`
файл со статистикой работы системы, а именно:
- cpu (нагрузка по ядрам 0, 1..)
- intr (прерываний всего, сюда не смотрим)
- btime (сейчас минус uptime)
- processes
- procs_running
- procs_blocked
- ctxt (смены контекста)
- softirq (intr?)

этот файл используется `mpstat`, `sar`, `top` и его вариациями

### `/proc/loadavg`
файл, содержащий очень важный показатель для мониторинга

сам по себе Load Average - среднее кол-во ожидающих ресурсов CPU процессов за один из трёх промежутков времени (1, 5, 15 минут)

полная нагрузка на одно ядро равна единице. 
eсли Load Average >= <cores_amount> - нагрузка на сервер слишком большая и что-то его перегружает.

### `/proc/sys`
файл интерфейса настройки параметров ядра, изменение которых осуществляет sysctl

пример: 
```
echo 1 > /proc/sys/net/ipv4/ip_forward 
```
включает маршрутизацию, но 
```
sysctl -w net.ipv4.ip_forward=1
```
делает то же самое

### `/proc/<pid>`
директория процесса по pid-у, содержащая:
- cmdline (команду процесса)
- status (Running, Sleeping, sTopped, Zombie)
- fd (file descriptor)
- maps (регионы памяти для pid)
- stat
