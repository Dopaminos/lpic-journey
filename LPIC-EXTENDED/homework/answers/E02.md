их значение рассматривалось в LPIC-1 (рассмотрено)

2: Да, совпадают, всё по нулям и там и там, что значит, что uptime получает loadavg из этого файла

3: `/proc/net/dev` выводит Interface + TX/RX пакеты, 
ip -s link так же выполняет эту функцию, но так же эта команда может описать интерфейс, указав все его параметры вроде значения MTU.

4: рассмотрим процесс 594 (sshd), 
смотрим /proc/594/cmdline и видим /usr/sbin/sshd -D
имя процесса можно увидеть, например через /proc/594/stat, где Name будет первым параметром

5: ls -l /proc/594/ покажет, чем является /proc/594/exe, здесь это /usr/sbin/sshd

# sysfs
1: То же, что и в третьем пункте procfs. В /sys/class/net устройства видны по интерфейсам, в виде директорий, внутри них есть папки со всеми их свойствами.

`ip link` - упрощённый вывод только самых нужных свойств
